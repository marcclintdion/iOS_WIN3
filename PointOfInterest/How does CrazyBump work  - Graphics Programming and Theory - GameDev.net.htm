you can use a sobel filter

here's some code to compute a tangent space normal map from a heightmap / grayscale image

float2 off = float2(1.0f, 1.0f) / srcTexDim;

	float s00 = tex2D(texHeight, In.UV + float2(-off.x,	-off.y)).x;
	float s01 = tex2D(texHeight, In.UV + float2( 0,		-off.y)).x;
	float s02 = tex2D(texHeight, In.UV + float2( off.x,	-off.y)).x;

	float s10 = tex2D(texHeight, In.UV + float2(-off.x,	0)).x;
	float s12 = tex2D(texHeight, In.UV + float2( off.x,	0)).x;

	float s20 = tex2D(texHeight, In.UV + float2(-off.x,	off.y)).x;
	float s21 = tex2D(texHeight, In.UV + float2( 0,		off.y)).x;
	float s22 = tex2D(texHeight, In.UV + float2( off.x,	off.y)).x;

	float sobelX = s00 + 2 * s10 + s20 - s02 - 2 * s12 - s22;
	float sobelY = s00 + 2 * s01 + s02 - s20 - 2 * s21 - s22;
	float sobelZ = 0.5f * sqrt(1.0f - sobelX * sobelX - sobelY * sobelY);

	float3 normal = normalize(float3(sobelX, sobelY, sobelZ));

	Out.Color = float4(normal * 0.5 + 0.5, 1);

However, crazybump must be doing something CRAZY to get these nice bump maps ;-) I suspect it involves some sort of adaptive blurring of the source image.
sourceforge.net/projects/megatexturedemo

If it was easy, they wouldn't be selling it for $300

The field of research is "computer vision". There's a lot of publications that explain exactly how to do this, but unfortunately, they almost always require two photos of the surface, either with the camera moved to two different locations, or with a static camera and two different lighting conditions. Generating reliable output from a single input image is very advanced (beyond the simple methods, like isolated color->height conversion).
